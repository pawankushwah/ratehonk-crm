### Technical Implementation

#### Database Schema
```sql
-- Use existing schema but ensure these fields exist:
invoices table:
- id, tenant_id, customer_id
- invoice_number, status, invoice_date, due_date
- subtotal, tax_amount, total_amount
- notes, created_at, updated_at

invoice_items table (if line items needed):
- id, invoice_id, description, quantity, rate, amount
```

#### API Endpoints Structure
```
GET    /api/invoices-v2              # List with pagination/filtering
POST   /api/invoices-v2              # Create new invoice  
GET    /api/invoices-v2/:id          # Get single invoice
PUT    /api/invoices-v2/:id          # Update invoice
DELETE /api/invoices-v2/:id          # Delete invoice

POST   /api/invoices-v2/import       # File upload and parsing
POST   /api/invoices-v2/import/preview # Preview parsed data
POST   /api/invoices-v2/import/confirm # Confirm and create invoices

GET    /api/invoices-v2/export       # Export invoices (filtered)
GET    /api/invoices-v2/:id/pdf      # Generate PDF for single invoice
```

- Use React with TypeScript

- React Query/TanStack Query for data fetching
- Lucide React for icons
- React Table for data tables
- File upload with drag & drop support
